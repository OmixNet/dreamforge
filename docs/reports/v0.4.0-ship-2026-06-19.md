## Lessons captured

1. **Rebrand string vs identifier 分 PR** (PR 16 rebrand + PR 18 migration). Each
   identifier family has different data-migration semantics, so each got
   different treatment.
2. **Per-family strategy** (PR 18): localStorage 3-layer migration, file token
   dual-recognize + migration script, URL param/sentinel direct rename. Risk
   follows data-migration risk.
3. **`\b` regex trap with CJK characters** (PR 20): Japanese/Korean text where
   `DreamForge` is followed by CJK chars — word boundary `\b` doesn't match.
   Second pass without `\b` caught 53 strings.
4. **`\b` regex trap with JSON escapes** (PR 21): en.json `\\n\\n` (literal
   backslash + n chars) — the trailing `n` is a word char, no boundary at `D`.
   Plain `str.replace` is the safer tool for user-visible string swaps.
5. **Code-level grep misses system integration** (PR 21): Info.plist,
   macOS permission dialogs, real GUI surfaces — only user GUI verify catches.
6. **Coverage gate polarity**: lock threshold at real measurement, not target.
   PR 13.4: actual 63.62, gate 63.5 (0.12pp buffer); prevents regression without
   chasing unreachable numbers.
7. **Test allowlist is cumulative** (PR 16 → 20 → 21): each rebrand surface
   added to the guard test, future rebrands auto-catch.
8. **Macos image pipeline without extra deps** (PR 19): sips + CoreSVG → PNG;
   iconutil → icns (requires `.iconset` suffix); PIL → ico. CoreSVG logs
   "CoreSVG has logged an error" but produces valid output.
