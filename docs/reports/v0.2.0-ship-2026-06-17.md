## Migration from v0.1.4 to v0.2.0

**No breaking changes** for existing v0.1 users:
- 5-entry slim sidebar unchanged
- 4-essential StatusBar unchanged
- dream CLI 4-tier path resolution unchanged
- localStorage keys unchanged (`dreamforge.dreamCliPath`, theme, etc.)
- AGPL-3.0-or-later license unchanged

**New localStorage keys** (added by PR 10/11.5, optional, with defaults):
- `dreamforge.llmBaseUrl` (default: `https://api.siliconflow.cn/v1`)
- `dreamforge.llmModel` (default: `deepseek-ai/DeepSeek-V4-Pro`)

**New env var** (required for cloud LLM):
- `DREAMFORGE_LLM_API_KEY` (in user's shell, not dreamforge settings)

**New macOS UserDefaults key** (required for cloud LLM to pass consent gate, in `com.OmixNet.dreamvault.gui` domain):
- `DreamVault.allowCloudSendRawSummary` = `true` (set via `defaults write com.OmixNet.dreamvault.gui DreamVault.allowCloudSendRawSummary -bool true`)

**New tauri command args** (PR 10):
- `dreamvault_status` / `dreamvault_run` / `dreamvault_report` accept new `llmBaseUrl` + `llmModel` args (both optional, no-op when empty)

6. **no `as any` / no `#[allow(...)]` / no `eslint-disable` in v0.2 code** (per Tolaria AGENTS.md tradition).
