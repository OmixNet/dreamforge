# v0.3.0 — Coverage ramp + gate baseline lock

**Ship date:** 2026-06-18
**Tag:** `v0.3.0` (annotated, points to the post-ramp commit)
**Scope:** Test coverage ramp-up across slim-mode hot paths, branches gate re-baselined, dreamforge pushed to the OmixNet remote for the first time.

This release is **test-only**: no new features, no breaking changes, no user-visible UX change. The v0.3.0 milestone exists to (a) lock the coverage gate at the real measurement so future changes cannot silently regress, and (b) start a slow, multi-PR push toward 70% branches over the rest of v0.3.

## What's in v0.3.0

### PR 12 — Initial coverage ramp + gate lock (60 → 62.97%)

Four commits on top of v0.2.0:

- **`d503fae` test: ramp coverage in PR 12 (v0.3 prep)** — Initial 5 new test files (dreamCliPath, LlmSettingsField, VaultDropdown, VaultSettingsSection, vaultListStore) + a one-line mock-handler fix (added `hidden_defaults: []` to `mockVaultList` so it matches the Rust `PersistedVaultList` shape exactly).
- **`c36d8ed` test: PR 12.1 branch ramp** — 28 new tests for 4 low-coverage modules (vaultErrors, domScroll, vault-dialog, useMenuEvents).
- **`9605220` test: PR 12.2 branch ramp** — 71 new tests across 5 modules (editorTitleHeadingText, folderActionUtils + useFolderDelete + useFolderRename, editorModePosition). Also adds `rename_vault_folder` / `delete_vault_folder` mock handlers so folder-action hooks' invoke paths resolve in non-Tauri test mode.
- **`e15d1b5` test: PR 12.3 branch ramp** — 13 new tests for `useInlineWikilinkSuggestionsState` + `editorParsedBlockPreload`.
- **`d32e79e` chore(coverage): lock branches gate at 62.5%** — Flips the gate polarity: the 60% v0.2 baseline was below the real measurement, which let dead-code branches silently drag the average down without tripping CI. Locking at the real value means any future change that drops branches below 62.5% must be addressed before merge.

### PR 13 — Second ramp + gate bump (62.5 → 63.5%)

Four commits on top of the lock:

- **`6212f88` test: PR 13 first slice** — 30 new tests across 5 modules (editorSchema, editorContentFlushRegistration, useRightPanelExclusion, useEditorModePositionSync, viewFilterArrayProperties).
- **`f1b7345` test: PR 13 second slice** — 12 new tests for `useNativePathDrop` + `useInlineWikilinkSelection`.
- **`e01f024` test: PR 13 third slice** — 11 new tests for sidebar pure helpers (`getPointerMenuPosition`, `getElementMenuPosition`, `computeReorder`, `applyCustomization`).
- **`c1b8f81` test: PR 13 final slice** — 14 new tests for `whiteboardPlatformPermissionRejection` + `appKeyboardShortcuts`.
- **`af93ac4` chore(coverage): bump branches gate to 63.5%** — Each new test file added ~0.1pp (diminishing returns). The remaining used code that would push toward 65 is either component-rendering (InlineWikilinkInput JSX with 47 uncovered branches) or pulls in many unrelated deps (useAppPreferences → useAiAgentPreferences / useTelemetry / useThemeMode / useDocumentThemeMode). The marginal cost of chasing 65 is now higher than the value.

### Push to remote

## Limitations

- **Used code is at ~84% branches, not 63%.** The Tolaria-residual files (AiWorkspace, ConflictResolver, PulseView, etc.) drag the average down — they survive the DREAMFORGE_SLIM deletes but aren't exercised in slim mode. A future PR 14/15 may selectively exclude the dead ones from the coverage include list, which would lift the gate measurement without lowering the bar.
- **PR 14 / 15 plan (from the gate comment)**: branches 63.5 → 64 → 65, each ~0.5-1pp. PR 14 only lands if a real used-code surface becomes worthwhile. The current cost of chasing 65 is high (mostly component-rendering tests with low branch density).
- **No GUI verify round.** macOS TCC blocks Mavis from opening the .app; user must run `pnpm tauri build` and verify `Settings → Vaults` + `VaultDropdown` manually. (Carry-over from v0.2.0 ship doc — same blocker.)

## Migration from v0.2.0

No code changes that affect runtime behavior. The two notable differences:

- Coverage gate is now **63.5% branches** (was 60% in v0.2). CI will fail if any future change drops branches below this number.
- Remote is configured: `git@github.com:OmixNet/dreamforge.git` (was no remote at v0.2.0).
