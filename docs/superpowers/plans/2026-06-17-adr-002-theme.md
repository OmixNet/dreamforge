# ADR-002: Theme — design tokens + 2 themes

- Status: Accepted (v0.1.3, 2026-06-17)
- Deciders: dreamforge core
- Source: PR 6 (StatusBar 4th essential = theme toggle), PR 8 (no theme library change), PR 9.5 (EditorTheme.css cleanup)

## Context

DreamForge uses 3 render libraries with separate theme systems:

- **BlockNote 0.46.2** (editor) — CSS-driven, no theme prop
- **Mantine 7** (UI components) — ThemeProvider + design tokens
- **Custom CSS** (`Editor.css` 521 lines, `EditorTheme.css` 382 lines post-cleanup) — 3-axis design tokens

v0.1 needs:

- Light + dark theme (Tolaria shipped both, expected by users)
- Theme toggle in StatusBar (4th essential per ADR-001)
- Theme persistence (localStorage)
- No per-component theme code (consistency)

## Decision

- **2 themes**: light (default) + dark, **no system-follow** (macOS prefers user-chosen)
- **3-axis design tokens**:
  - color (primary / secondary / tertiary + bg / fg / border variants)
  - spacing (4 / 8 / 16 / 24 / 32)
  - typography (12 / 14 / 16 / 20 / 24)
- **Tokens defined in `EditorTheme.css` as CSS custom properties** (`--dreamforge-color-primary` etc.)
- **Component code reads `var(--token)`** — no per-component theme props
- **Mantine `ThemeProvider`** provides the same tokens to Mantine components (block-level theme sync)
- **BlockNote** uses its own CSS but maps to same color tokens (override in `EditorTheme.css`)
- **Persistence**: localStorage `dreamforge.theme` = "light" | "dark"
- **Switch trigger**: StatusBar theme button + Settings

## Consequences

### Positive

- Theme change = 1 CSS class toggle on root, all 3 libraries re-render with new tokens
- No `<ThemeProvider>` prop drilling
- `EditorTheme.css` 661 → 382 lines (PR 9.5 cleanup): removed mermaid / tldraw / katex residual rules (-42%)

### Negative / Lost features

- No system-follow (`prefers-color-scheme` — was in Tolaria)
- No high-contrast mode
- No custom user themes (v0.2+ feature)

### Forward compatibility

- Custom themes (v0.2+): user can add a third theme by adding CSS variables + `ThemeProvider` entry
- Theme persistence schema: `dreamforge.theme` string key, additive migration

## Alternatives Considered

- **Theme library (next-themes, theme-ui)**: extra dep, no value over CSS variables. Rejected.
- **System-follow (`prefers-color-scheme`)**: macOS users want explicit choice, not auto. Rejected.
- **3+ themes in v0.1** (sepia, high-contrast): scope creep. Rejected, v0.2+.

## Implementation pointer

- Theme toggle: `src/components/status-bar/ThemeToggle.tsx`
- Persistence: `src/lib/themeStorage.ts` (localStorage read/write)
- CSS tokens: `src/components/EditorTheme.css` L1-L100 (post-PR 9.5 cleanup)
- Mantine sync: `src/components/AppMantineProvider.tsx`
